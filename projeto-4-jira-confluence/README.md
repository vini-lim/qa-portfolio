# Projeto 4 — Gestão de Testes | Jira + Confluence

Este projeto faz parte do meu portfólio de QA e tem como objetivo demonstrar
o uso do ecossistema **Atlassian (Jira + Confluence)** para gestão de testes,
rastreamento de bugs e documentação de qualidade — simulando o fluxo real
de trabalho de um Analista de QA em um time de desenvolvimento.

---

## 🎯 Objetivo

Demonstrar conhecimento prático nas ferramentas Jira e Confluence, aplicando
conceitos reais de gestão de qualidade: criação de épicos, histórias de usuário,
tasks de QA, bug reports, sprints e documentação técnica.

## 🔄 Fluxo de Trabalho Simulado

O projeto simula o ciclo completo de trabalho de um QA em um time Scrum:
```
Épico → História de Usuário → Task de QA → Execução → Bug Report → Sprint Review
```

1. **Épicos** criados para cada grande funcionalidade
2. **Histórias de usuário** com critérios de aceite definidos
3. **Tasks de QA** representando o trabalho técnico (planejar, escrever, executar)
4. **Execução** simulada movendo cards pelo board Scrum
5. **Bug reports** criados e vinculados às histórias correspondentes
6. **Documentação** registrada no Confluence com planos e relatórios

---

## 🧠 Conceitos Demonstrados

**Jira:**
- Criação e organização de épicos
- Escrita de histórias de usuário com critérios de aceite
- Criação de subtarefas de QA
- Bug reports com severidade, prioridade e passos de reprodução
- Gerenciamento de sprint (criação, execução e organização do board)
- Vínculo entre tickets (bug → história → épico)

**Confluence:**
- Criação de espaço de documentação
- Hierarquia de páginas (espaço → seção → página)
- Plano de testes documentado
- Relatório de execução com métricas
- Bug reports detalhados com passos de reprodução

**Metodologia:**
- Scrum (épicos, histórias, sprint, board)
- BDD conceitual (critérios de aceite nas histórias)
- Rastreabilidade (bug vinculado à história vinculada ao épico)
